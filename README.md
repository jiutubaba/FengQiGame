# 风起游戏

“风起游戏”归属于湖北风起文化有限公司。本项目是面向正式运营的游戏管理平台：前端使用 React + Vite，API 使用 Express，业务数据使用 PostgreSQL；生产部署由 Docker Compose 管理 PostgreSQL、应用服务和 Caddy 反向代理。

系统不会预置演示地图、玩家或弱口令账号，也不会把参考站登录凭据写入项目。首次启动时由部署人创建本系统自己的管理员。

## 快速启动

Windows 双击 `启动网站.bat`。脚本会自动选择可用模式：Docker 引擎正常时使用完整容器部署；Docker 暂不可用但本机 PostgreSQL 可用时使用原生模式：

1. 首次运行会要求填写站点地址、管理员用户名和至少 12 位的强密码。
2. 脚本生成不纳入版本控制的 `.env`，数据库密码使用系统随机数生成。
3. Docker 模式构建数据库、应用和反向代理；原生模式创建最小权限数据库角色并启动 Node 服务。
4. 浏览器打开 `SITE_ADDRESS`。以后再次双击只会更新并启动服务，不会覆盖 `.env`。

原生模式仅监听 `http://127.0.0.1:3000`，适合当前电脑在无法启动 Docker Desktop 时继续开发和内网验收；正式公网部署仍应使用 Docker + Caddy + HTTPS。原生模式初始化管理员后不会把管理员明文密码保留在 `.env`。

正式域名必须先把 DNS 指向服务器，并在首次配置时填写 `https://实际域名`。Caddy 会自动申请和续期 HTTPS 证书。

## 权限模式

- 系统管理员：管理全部地图、账号、角色、地图功能权限、系统设置和审计日志；只有管理员能创建/归档地图及清理运行数据。
- 普通用户：只看到管理员明确授权的地图；每张地图继续按指标、配置、玩家、排行榜、风控、礼包、主播、埋点、日志、文件和 API Key 分别授权。
- 游戏客户端：不使用后台账号。每个 API Key 固定到一张地图和一个环境，并按写入玩家、排行榜、风险事件、日志、指标、埋点、读取消息、领取礼包分别授权。
- 公开群抽参与者：只可通过不可猜测的活动链接报名和查看已公开的开奖结果，不能访问后台接口。

## 已打通功能

- 安全会话登录、退出、个人资料、修改密码和登录限流
- 管理员账号创建、停用、重置密码、地图级功能权限矩阵
- 地图创建、编辑、归档；正式服/测试大厅/测试服数据隔离
- 指标趋势、玩家增删改查和三类封禁状态
- 可配置实时排行榜、前 100 名发布快照、历史榜单与榜单封禁联动
- 风控规则、客户端幂等事件上报、风险证据和玩家三类封禁处置
- 游戏内消息发送、客户端拉取与确认送达
- 礼包定义、批量发放、客户端领取与确认
- 群抽创建、公开报名、随机开奖、中奖名单公开
- 主播、埋点、聚合日志、配置 JSON、预加载代码
- 文件夹、真实文件上传/下载/删除、大小限制和可执行脚本拦截
- 游戏客户端 API Key 最小权限、一次性 Token 展示和停用
- 管理员危险数据清理、全量审计日志、系统状态与站点设置
- PostgreSQL 迁移、健康检查、容器部署、备份和人工确认恢复

## 开发与验证

```powershell
npm ci
npm run ai:docs:check
npm test
npm run build
npm run audit:prod
```

长期开发开始前运行 `npm run ai:context -- <源码路径>` 获取必读治理文档；路由、权限、迁移或环境变量变化后运行 `npm run ai:docs:build` 刷新自动事实。完整规则见 [AI 开发治理入口](.ai/README.md)。

数据库集成测试必须指向隔离的本机 PostgreSQL，禁止使用正式库：

```powershell
$env:NODE_ENV='test'
$env:DATABASE_URL='postgres://fengqi_test:密码@127.0.0.1:55432/fengqi_test'
npm run test:integration
```

## 运维入口

- `运维/备份数据.ps1`：生成 PostgreSQL 自定义格式备份，默认保留 14 天。
- `运维/恢复数据.ps1 -BackupPath <文件>`：要求输入数据库名确认，自动生成安全备份，停止应用后恢复并等待健康检查通过。
- `运维/备份上传文件.ps1`：停止写入后快照 `uploads_data` 或本机 `uploads/`，默认保留 14 天。
- `运维/恢复上传文件.ps1 -SnapshotPath <文件>`：校验归档路径，自动生成安全快照后恢复上传文件并等待应用健康。
- `运维/检查部署.ps1`：重新安装锁定依赖，执行单元测试、构建、生产依赖审计和 Compose 配置检查。

完整说明见 [部署与运维](docs/部署与运维.md)、[游戏客户端接入](docs/游戏客户端接入.md) 和 [上线验收清单](docs/上线验收清单.md)。

`public/data/` 是随项目部署的 XYPlugin/XYWE 文档与版本档案，不包含后台账号、玩家或运营业务数据；工具安装包未被伪造为可下载资源，应由管理员上传到受控文件空间后再发布。
