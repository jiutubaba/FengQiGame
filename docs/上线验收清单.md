# 上线验收清单

## 基础设施

- [ ] 正式域名 DNS 已生效，浏览器证书有效且只通过 HTTPS 登录
- [ ] 服务器只开放 80/443，PostgreSQL 没有公网端口
- [ ] `.env` 权限受限，管理员和数据库使用独立强密码
- [ ] `docker compose ps` 三个服务均为 healthy/running
- [ ] `/api/system/health` 返回 `status: ok`

## 账号与权限

- [ ] 初始管理员已登录并立即确认个人资料
- [ ] 创建一个普通用户，只授权一张测试地图和一个功能
- [ ] 普通用户看不到未授权地图、管理员菜单和未授权功能
- [ ] 停用普通用户后，原会话立即失效
- [ ] 重置密码后，该用户全部会话立即退出

## 地图与客户端链路

- [ ] 正式服、测试大厅、测试服各自写入同 UID，不互相覆盖
- [ ] 客户端上报玩家、指标、日志和埋点后，后台数据正确
- [ ] 后台发送消息，客户端读取、写入成功后 ACK，后台状态为已送达
- [ ] 后台发放礼包，客户端领取成功后 ACK，不会重复发奖
- [ ] 每个环境使用独立 API Key，停用后请求立即返回 401

## 业务功能

- [ ] 地图配置保存后刷新页面仍一致，JSON 语法错误不能保存
- [ ] 玩家封禁状态、礼包、主播、埋点和日志增删改均写入审计日志
- [ ] 客户端批量写入实时榜，后台发布快照后历史名次不再被后续数据改写
- [ ] 榜单封禁玩家不会进入实时榜和新快照，解除封禁后可重新参与排名
- [ ] 风险事件使用同一 `eventId` 重试不会重复入库，处置结果能联动三类玩家封禁
- [ ] 普通用户只有查看权限时不能创建榜单、规则或处置风险事件
- [ ] 群抽公开链接可报名，同一 UID 不能重复，开奖后显示中奖名单
- [ ] 文件上传、下载、文件夹和删除有效；超大文件及脚本文件被拒绝
- [ ] 管理员运行数据清理要求输入地图名，且不会删除地图配置和文件

## 恢复能力

- [x] 执行一次数据库备份并确认 `.dump` 文件非空
- [x] 在隔离 Docker 环境成功恢复数据库备份并核对回滚结果
- [x] 上传文件卷有独立快照与恢复脚本，并在隔离 Docker 卷完成回滚验证
- [ ] 独立私有 OSS 已启用私有 ACL、阻止公共访问、AES-256、14 天对象过期和 1 天分片清理
- [ ] 最小权限 RAM 凭证只能访问正式备份 Bucket 的 `prod/` 前缀，删除与跨 Bucket 访问被拒绝
- [ ] 正式 systemd timer 每天北京时间 03:30 运行，数据库与上传卷异机副本 CRC64 一致
- [ ] 失败告警测试邮件由已确认负责人收到，OSS 远端副本在隔离环境实际恢复成功

任何未勾选项都应记录负责人、计划时间和风险说明；未完成恢复演练前不建议承诺生产数据可恢复。

- 本机原生模式已覆盖生产构建、迁移幂等、隐藏进程启动、受控重启、健康检查、静态资源、登录退出及备份恢复。
