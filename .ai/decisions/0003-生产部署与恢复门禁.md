# ADR-0003：生产部署与恢复门禁

- 状态：已接受
- 日期：2026-07-14

## 背景

项目支持 Docker 生产拓扑和无法启用虚拟化时的本机原生模式。两者的安全与可恢复能力不同。

## 决策

- 公网生产只接受 Docker Compose、Caddy HTTPS、数据库不暴露公网端口的拓扑。
- 原生模式只用于本机开发和内网验收，不作为公网发布替代方案。
- 数据库备份和上传卷快照分别管理；未在隔离环境恢复成功前不得声明具备恢复能力。
- 未实际完成的证书、容器与恢复项目必须保留在验收清单和 backlog。

## 后果

- 代码构建和本机测试通过不等于生产上线完成。
- 发布流程必须保留备份、健康检查、业务冒烟和恢复证据。

## 验证

按 `docs/上线验收清单.md` 完成人工勾选并记录负责人、时间与证据。
