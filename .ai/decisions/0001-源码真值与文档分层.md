# ADR-0001：源码真值与文档分层

- 状态：已接受
- 日期：2026-07-14

## 背景

项目已具备面向部署和接入的 `docs/`，但缺少适合多次 AI 开发的源码路由、业务不变量和可检查事实。复制现有文档会造成双重维护。

## 决策

- 真值顺序固定为源码/迁移/配置、自动生成事实、系统文档、会话与 backlog。
- `docs/` 面向使用和运营，`.ai/` 面向开发治理；两者引用而不复制。
- 路由、权限、迁移、环境变量和测试入口由工具生成，生成文件禁止手改。
- 稳定事实进入系统文档，长期取舍进入 ADR，近期过程滚动保留 5 条。

## 后果

- 行为变化需要同步代码、事实和系统文档。
- 自动检查会阻止生成事实漂移和文档结构损坏。
- Git 历史承担长期变更追踪，不无限堆积会话摘要。

## 验证

运行 `npm run ai:docs:build` 后连续两次输出一致，`npm run ai:docs:check` 通过。
