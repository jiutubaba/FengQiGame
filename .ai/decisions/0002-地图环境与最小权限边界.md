# ADR-0002：地图、环境与最小权限边界

- 状态：已接受
- 日期：2026-07-14

## 背景

系统同时服务多张地图、三个运行环境、后台管理员、受限运营用户、游戏客户端和公开群抽参与者。边界混用会造成越权或数据污染。

## 决策

- 地图是租户边界，运行数据按 `map_id + environment` 隔离。
- 管理员、普通用户、API Key 和公开 Token 使用互不替代的身份链路。
- 普通用户必须同时拥有 `map.view` 和功能权限；API Key 固定地图、环境和最小 `game.*` 权限。
- 前端显隐不承担授权，所有写操作由服务端校验并按风险写审计。

## 后果

- 任何新增业务表和接口都要明确地图、环境、身份与权限语义。
- 跨环境聚合必须作为显式新能力设计，不能通过省略过滤条件实现。

## 验证

隔离集成测试覆盖管理员、普通用户、API Key、三环境同 UID 和越权失败场景。
