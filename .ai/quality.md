# 质量与验收

## 检查层级

1. `npm run ai:docs:check`：治理文件、路径、链接、生成事实、会话上限和敏感信息。
2. `npm test`：安全工具和输入校验单元测试。
3. `npm run build`：生产前端构建。
4. `npm run audit:prod`：生产依赖漏洞审计。
5. `npm run test:integration`：隔离 PostgreSQL 全链路测试。
6. 按 `docs/上线验收清单.md` 执行部署、权限、客户端、备份恢复和浏览器验收。

`npm run check` 必须包含 AI 文档检查、单元测试和生产构建，是日常改动的最低门禁。

## 行为变更验收

- API：成功/失败格式、状态码、授权、校验、事务和审计均有证据。
- 数据：迁移可从空库顺序执行且重复启动不重放；环境与地图隔离不被破坏。
- 前端：管理员、受限用户、空状态、错误状态、桌面与窄屏均按风险比例验证。
- 运维：启动、健康检查、备份和恢复不得只凭静态脚本宣称通过。

## 结果表述

- 已执行且通过：写明命令和结果数量。
- 已修改但未运行：明确写“已修改，未测试”。
- 受外部条件阻塞：记录条件、已完成替代验证和仍需人工完成的验收项。
- 同一问题连续失败两次后暂停，复核假设，不继续堆叠绕行方案。
