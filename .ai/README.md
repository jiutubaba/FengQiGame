# 风起游戏 AI 开发治理入口

更新时间：2026-07-14

## 用途

`.ai/` 保存能让开发者与 AI 长期、连续、安全维护本项目的事实、规则和决策。面向部署者和运营人员的操作说明仍位于 [`docs/`](../docs/)。

## 真值层级

发生冲突时按以下顺序处理，不得用低层文档覆盖高层事实：

1. 源码、数据库迁移、运行配置和自动化测试。
2. `.ai/generated/` 中由源码确定性生成的事实快照。
3. `.ai/systems/`、架构、安全、质量和运维文档。
4. ADR、近期会话与 backlog。
5. 旧版后台、参考站截图或口头经验。

发现冲突时先核对源码；确认文档过期后同步修正文档和生成事实，并在会话归档中记录。

## 阅读顺序

1. 阅读项目根目录 [`AGENTS.md`](../AGENTS.md)。
2. 阅读本文件和 [`router.md`](router.md)。
3. 运行 `npm run ai:context -- <目标路径>`。
4. 按命令输出阅读系统文档和 ADR，再开始修改。

## 目录职责

- `architecture.md`：系统边界、组件关系和主要数据流。
- `router.md`：源码路径到必读上下文和验证命令的路由。
- `code_style.md`：代码组织与可维护性规则。
- `security.md`：认证、授权、数据和文件安全不变量。
- `quality.md`：测试层级、验收标准和失败报告规则。
- `operations.md`：开发、部署、备份恢复的工程边界。
- `systems/`：按业务域维护的当前系统事实。
- `decisions/`：长期架构决策记录。
- `generated/`：生成工具输出，禁止手改。
- `backlog.md`：有证据、可验收的未完成工作。
- `sessions.md`：最近 5 次重要任务归档。

## 维护规则

- 只记录已从源码、测试或正式运维结果证实的内容；未验证事项必须写入 backlog 并标注证据与状态。
- 小型实现细节不需要进入系统文档；影响接口、数据、权限、失败语义或运维流程的变化必须同步。
- 自动事实通过 `npm run ai:docs:build` 刷新，通过 `npm run ai:docs:check` 检查漂移。
- 任何文档不得包含 `.env` 实值、完整 Token、密码、生产玩家信息、证照图片或备份内容。
