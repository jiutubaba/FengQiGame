# 安全不变量

更新时间：2026-07-14

## 凭据与会话

- 密码使用安全哈希；会话与 API Key 只保存哈希，完整 Token 不落库。
- 登录有限流；停用用户、重置密码和修改密码必须使既有会话失效。
- Cookie 的 Secure 策略由环境决定；公网生产必须 HTTPS、`COOKIE_SECURE=true`、正确配置 `TRUST_PROXY`。
- 日志必须脱敏 Cookie、Authorization、`FQ-Map-Key` 和密码字段。

## 授权

- 管理员路由使用服务端 `requireAdmin`；普通用户必须同时拥有 `map.view` 和目标功能权限。
- API Key 固定 `map_id`、`environment` 和权限数组；每个客户端接口必须检查独立 `game.*` 权限。
- 前端路由保护、菜单显隐和禁用状态不是授权依据。

## 请求与数据

- 非只读浏览器请求执行同源 Origin 校验；游戏客户端路径使用 API Key，不依赖浏览器 Origin。
- JSON、表单和上传大小均有限制；公开报名接口限流并阻止同活动 UID 重复。
- 文件路径必须标准化并限制在地图上传根目录；拒绝脚本和可执行扩展名，下载前核对元数据所属地图。
- 不得在错误响应中暴露 SQL、堆栈、凭据或内部路径；保留 `requestId` 便于排障。

## 测试与文档数据

- 集成测试要求 `NODE_ENV=test`、数据库主机为 `127.0.0.1`、数据库名为 `fengqi_test`，否则立即终止。
- `.ai/`、`docs/`、测试夹具和 Git 历史不得出现真实密码、完整 API Key、生产玩家资料、营业执照图片或备份内容。
- 旧参考站账号和域名不属于本系统凭据，不得复制进入项目。
