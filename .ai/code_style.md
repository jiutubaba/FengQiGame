# 代码风格与组织

## 基本原则

- 命名直接表达职责，避免 `utils2`、`common-new`、`temp` 等泛名。
- 优先让一个业务流程在相邻代码中完整可读；只有确有复用或边界价值时抽取 helper。
- 保持现有 ESM、React 函数组件、Express Router、Zod 校验和参数化 SQL 风格。
- 不引入与需求无关的抽象、兼容层、回退分支或新依赖。
- 长文件修改使用小补丁，先定位真实调用链，避免覆盖同文件中的其他有效改动。

## 前端

- 页面不得用硬编码样本替代 API 数据；加载、空状态、错误和权限不足必须可区分。
- 权限菜单与按钮显隐复用服务端返回权限，但不能被视为安全校验。
- 全站视觉保持 Attio 式温暖白与雾灰画布、黑色精密文字和单一低饱和蓝色强调；常规业务区以留白、1px 边界、表格和列表建立层级，避免渐变光晕、重阴影与无意义卡片墙。响应式断点修改必须检查桌面与窄屏。
- 全站界面字体通过 Fontsource 本地托管：业务正文与控件使用 `Noto Sans SC Variable`，品牌展示文字可使用 `Noto Serif SC Variable`；运行时不得依赖在线字体服务。
- 业务正文和控件默认使用 13–14px；常用导航和页面说明优先使用 14px，辅助说明优先使用 12–13px，11px 仅用于编号、技术元数据等弱层级信息；数字指标使用等宽数字，ID、权限名与技术字段使用等宽字体，窄屏主要点击区域不得小于 44px。
- 颜色、字号、间距、6–12px 圆角、阴影、焦点环和动效统一复用 `styles.css` 设计令牌；交互控件必须具备键盘焦点状态。页面进入、行与按钮反馈、抽屉与模态框使用约 140–200ms 的统一节奏，并为 `prefers-reduced-motion` 提供降级。
- 登录后工作台采用均衡紧凑密度：页面标题约 26px，页面区块间距 18–20px，工具栏约 50–52px，桌面控件 36–40px，表格表头约 40px、数据行 46–48px；移动端控件仍不得低于 44px。地图中心卡片网格按宽度使用 4/3/2/1 列，封面控制在约 136–158px，占位封面使用浅灰网格和居中小标记，列表行控制在约 120px。
- `MapWorkspace.jsx` 承载多业务域；新增复杂功能前先评估是否形成独立组件，禁止继续无边界堆叠。

## 服务端与数据库

- 所有外部输入先经 Zod 或等价明确校验；SQL 必须参数化。
- 多表原子操作使用 `transaction`；错误使用统一 `HttpError`/错误处理中间件。
- 新表或字段只通过递增迁移新增；已发布迁移不得原地改写。
- 变更权限、路由、环境变量或迁移后必须刷新自动事实。

## 文档与注释

- 注释解释业务原因、边界或危险点，不复述语句本身。
- 稳定合约写入对应 `systems/` 文档；长期取舍写 ADR；一次性过程只进入会话归档。
- `.ai/generated/` 只能由工具生成。
