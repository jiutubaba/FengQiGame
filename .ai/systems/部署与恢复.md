# 部署与恢复

更新时间：2026-07-15

操作步骤以 `docs/部署与运维.md` 为准，本文件记录开发不能破坏的部署合约。

## 生产拓扑

- Compose 管理 PostgreSQL、应用和 Caddy；数据库使用命名卷且不发布公网端口。
- Caddy 使用 `SITE_ADDRESS` 申请/续期证书并反向代理应用；正式站点必须 HTTPS。
- 应用启动先等待数据库健康，再按顺序执行迁移；仅当数据库完全没有用户时创建初始管理员。
- `.env` 由首次配置生成且不进入 Git，数据库密码与 `DATABASE_URL` 必须一致。

## 原生模式

- 当 Docker 不可用时，本机脚本创建最小权限 PostgreSQL 角色并启动隐藏 Node 进程。
- 只监听 `127.0.0.1:3000`；无 Caddy 自动 HTTPS，不允许直接暴露公网。
- 管理员明文密码和 PostgreSQL 管理员密码不得保留在生成的 `.env`。

## 备份与恢复

- PostgreSQL 使用自定义格式备份并按保留期清理；恢复要求输入目标数据库名确认，并由脚本强制生成恢复前安全备份。
- 数据库恢复先停止应用写入；恢复失败时应用保持停止，恢复成功后必须等待应用健康检查通过。
- 上传卷不在数据库备份中；`备份上传文件.ps1` 与 `恢复上传文件.ps1` 提供独立快照、路径安全校验、恢复前安全快照和恢复后健康检查。
- 两类备份脚本支持可选 `-PassThru` 返回产物；`备份到OSS.ps1` 负责上传到私有 OSS `prod/YYYY/MM/DD/`、请求 AES-256 加密、比较本地与远端 CRC64，并在失败时调用服务器 `msmtp` 告警。
- Linux 正式调度模板位于 `运维/systemd/`，固定每天 `19:30 UTC`（北京时间 03:30）运行并启用持久补跑；OSS 与邮件凭据只允许存在于服务器 root 可读配置文件。
- 本机 `backups/` 不是异机备份；正式环境必须配置异机或对象存储副本、调度、告警和负责人，并以实际恢复演练验收。
