# 排行榜与风控

更新时间：2026-07-14

## 排行榜

- 排行榜定义按地图、环境和稳定 `leaderboard_key` 唯一，配置名称、数值名称、升降序和启用状态。
- 实时条目按榜单与玩家 UID upsert，记录玩家名、游戏等级、分数、游戏次数和元数据。
- 查询实时榜时排除当前环境中 `rank_ban=true` 的玩家。
- 发布快照在事务内复制当时符合条件的前 100 名；快照条目保存独立名次和数值，后续实时更新、删除或封禁不能回写历史。
- 创建、修改、删除、发布和人工移除条目均写审计。

## 风控

- 风控规则按地图、环境和稳定 `rule_key` 唯一，包含名称、严重级别和启用状态。
- 客户端只能按已启用规则上报；`map_id + environment + event_key` 唯一，网络重试保持幂等。
- 事件保存上报时的规则名和严重级别快照、玩家、次数、证据 JSON、状态及处置人。
- 处置可以更新事件状态，并在同一事务内联动物品、存档、榜单封禁；玩家尚不存在时仍可保留事件，但不能伪造玩家记录。
- 规则删除使用外键定义的保留语义；历史事件仍保留上报时的规则 Key 和名称。

## 安全与验证

- 只有查看权限的普通用户不能创建榜单、发布快照、管理规则或处置事件。
- 证据 JSON 只保存处置所需结构化数据，禁止凭据、完整 Token 和无关个人信息。
- 验证幂等上报、升降序、封禁排除、快照不可变、三类封禁联动和审计记录。

主要证据：`server/db/migrations/003_leaderboards_risk.sql`、`server/routes/maps.js`、`server/routes/game.js`。
