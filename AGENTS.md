# 风起游戏项目指令

本文件是本仓库的项目级开发治理入口。适用于人工开发者和所有 AI 工具。

## 语言与沟通

- 用户可见内容、计划、验收报告和项目文档使用简体中文。
- 代码标识、权限名、API 路径、环境变量和原始错误保持源码原文。

## 开始任务前

1. 先读 `.ai/README.md` 和 `.ai/router.md`。
2. 对目标源码运行 `npm run ai:context -- <路径>`，读取输出的必读文档。
3. 若仓库已有未提交改动，先识别并保护，不得覆盖或回退用户修改。
4. 以源码、数据库迁移和配置为最高真值；旧版后台与外部参考站只能作为参考证据。

## 不可破坏的边界

- 地图业务数据必须同时按 `map_id` 和 `environment` 隔离；环境仅允许 `release`、`lobby`、`test`。
- 普通后台用户必须经过地图级最小权限校验；管理员权限不能被前端显隐逻辑替代。
- 游戏客户端只使用绑定地图和环境的 API Key；服务端只保存 Token 哈希，完整 Token 只在创建时返回一次。
- 会话 Token、API Key、密码、生产玩家数据、营业执照图片和数据库备份不得写入源码、文档、日志或 Git。
- 风险事件 `eventId` 保持幂等；排行榜历史快照不可被实时数据或后续封禁回写。
- 危险操作必须保留服务端授权、明确确认、审计记录和失败回滚边界。
- 集成测试只能连接显式隔离的本机测试库，禁止连接正式库或共享开发库。

## 修改与同步

- 业务行为或不变量变化：同步更新 `.ai/systems/` 对应文档。
- 路由、权限、迁移、环境变量或测试入口变化：运行 `npm run ai:docs:build`，不得手改 `.ai/generated/`。
- 长期架构取舍：新增 ADR；不得改写已生效 ADR 的历史结论，只能标记替代或废弃。
- 有源码证据的未完成工作：记录到 `.ai/backlog.md`；不得把猜测写成技术债。
- 完成重要任务：更新 `.ai/sessions.md`，滚动保留最近 5 条；稳定结论必须下沉到系统文档。
- `docs/` 面向部署与运营，`.ai/` 面向开发治理；两者通过链接关联，不复制整篇内容。

## 验证闭环

- 文档与事实：`npm run ai:docs:check`
- 单元测试与生产构建：`npm run check`
- 生产依赖：`npm run audit:prod`
- 数据库链路：仅在隔离 PostgreSQL 上运行 `npm run test:integration`
- 部署验收：按 `docs/上线验收清单.md` 执行；未实际完成的 Docker、HTTPS、备份恢复项目必须如实标记。
